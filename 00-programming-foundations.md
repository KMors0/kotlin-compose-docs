Если вы никогда не программировали — начните отсюда. После этой главы остальные разделы документации перестанут быть «магическими заклинаниями» и станут логичными инструментами.

## 0. Что такое программа?

Программа — это **последовательность инструкций для компьютера**. Компьютер сам по себе глупый — он умеет только выполнять команды одну за другой. Программа — это текст, который говорит компьютеру: «сделай это, потом это, потом если условие — сделай вот это, иначе вот это».

Аналогия: **рецепт блюда**. Рецепт — это программа для повара. В нём написано: «возьми 200 грамм муки, добавь 2 яйца, перемешай, если тесто жидкое — добавь ещё муки, выпекай 30 минут при 180 градусах». Повар (как компьютер) просто идёт по шагам.

Программирование — это **умение писать такие рецепты для компьютера**. Разница в том, что:
- Компьютер понимает только очень простые команды (сложить два числа, проверить равенство, прочитать из памяти).
- Поэтому сложные действия нужно **разбивать** на простые шаги.
- У компьютера нет «здравого смысла» — он сделает **ровно то, что написано**, даже если это бессмысленно.

### Зачем нужны языки программирования?

Процессор компьютера понимает только **машинный код** — последовательность нулей и единиц, типа `10110000 01100001`. Писать так невозможно для человека.

Поэтому придумали **языки программирования** — текст, который человек может читать, и который специальная программа (**компилятор**) переводит в машинный код. Kotlin — один из таких языков.

```
Kotlin код:  val x = 5 + 3
                    ↓ (компилятор переводит)
Машинный код: 10110000 00000101 00000011 00001000
                    ↓ (процессор выполняет)
Результат: x = 8
```

> [!TIP]
> **Компилятор** — это программа, которая читает ваш Kotlin-код и переводит его в формат, понятный процессору. У Kotlin есть свой компилятор (K2 — актуальная версия с Kotlin 2.0). Для разных платформ (Android, iOS, Web) компилятор генерирует разный машинный код — поэтому одна и та же Kotlin-программа может работать везде.

## 1. Переменная — это «коробка с именем»

Представьте: у вас есть коробка, вы написали на ней маркером «возраст», положили внутрь число 25. Позже можете достать это число, посмотреть его, заменить на 26. Это и есть **переменная**.

В Kotlin:

```kotlin
var age = 25       // создали коробку "age", положили 25
println(age)       // заглянули в коробку — выведет 25
age = 26           // заменили содержимое на 26
println(age)       // выведет 26
```

**Зачем нужны имена?** Компьютер работает с адресами в памяти (например, «ячейка 0x7F4A»), но людям такие числа неудобны. Имя переменной — это **человекочитаемая метка** для ячейки памяти. Когда вы пишете `age`, компилятор сам подставляет правильный адрес.

### `val` vs `var` — постоянная и изменяемая коробка

Kotlin разделяет переменные на два типа:

```kotlin
val year = 2026       // val — неизменяемая: положили и закрыли крышку с клеем
year = 2027           // ❌ ОШИБКА КОМПИЛЯЦИИ — val нельзя переназначить

var temperature = 20  // var — изменяемая: можно менять сколько угодно
temperature = 22      // ✅ ОК
temperature = 18      // ✅ ОК
```

> [!TIP]
> **`val` (value)** — переменная, которую можно назначить только один раз. Аналог — коробка, которую заклеили скотчем: открыл, положил, закрыл навсегда. Если содержимое — объект, вы можете изменять его внутренности (например, добавлять элементы в список), но не можете заменить весь объект на другой.
>
> **`var` (variable)** — переменная, которую можно переназначать сколько угодно раз. Аналог — обычная коробка: открывай, меняй содержимое, закрывай.
>
> **Почему Kotlin поощряет `val`?** Неизменяемые переменные проще понимать: если `val age = 25`, вы знаете, что в любой точке программы `age` равно 25. С `var` приходится отслеживать все места, где переменная может измениться. Меньше `var` → меньше багов.

### Имена переменных

Имя переменной должно быть **осмысленным**. `x`, `y`, `temp` — плохие имена (если это не математические координаты). `userAge`, `taskTitle`, `isLoggedIn` — хорошие: из имени понятно, что внутри.

```kotlin
// ❌ Плохо — непонятно, что это
val a = 25
val s = "Alice"
val f = true

// ✅ Хорошо — имя объясняет смысл
val userAge = 25
val userName = "Alice"
val isLoggedIn = true
```

Время, потраченное на придумывание хорошего имени, окупается: через полгода вы откроете код и сразу поймёте, что к чему.

## 2. Тип данных — это «контракт» о том, что лежит в коробке

Переменная хранит **значение**. Значения бывают разные: числа, текст, логические значения (истина/ложь), списки. **Тип данных** — это ярлык на коробке, который говорит, **какого рода** данные внутри.

Зачем это нужно? Компьютеру нужно знать, что делать с содержимым. Если в коробке число — его можно умножить. Если текст — его можно написать с заглавной буквы. Если попытаться умножить текст на число — это бессмыслица, и компилятор заранее (до запуска) скажет: «так делать нельзя».

### Основные типы в Kotlin

| Тип | Что хранит | Пример |
|-----|-----------|--------|
| `Int` | Целое число (от -2³¹ до 2³¹-1) | `42`, `-7`, `0` |
| `Long` | Большое целое (от -2⁶³ до 2⁶³-1) | `9_000_000_000L` |
| `Double` | Число с плавающей точкой | `3.14`, `-0.001` |
| `Float` | Меньшая точность, чем Double | `2.5f` |
| `Boolean` | Логическое значение | `true`, `false` |
| `String` | Текст (строка символов) | `"Привет"`, `"123"` |
| `Char` | Один символ | `'A'`, `'7'`, `' '` |

```kotlin
val age: Int = 25                // число
val name: String = "Alice"       // текст
val isAdult: Boolean = age >= 18 // логическое значение (true/false)
val height: Double = 1.75        // число с дробной частью
```

> [!TIP]
> **Двоеточие `:` после имени переменной** — это аннотация типа. `val age: Int = 25` читается как «объявить переменную `age` типа `Int` со значением 25». Часто тип можно не писать — компилятор сам догадается по значению: `val age = 25` (компилятор видит 25 и понимает, что это `Int`). Это называется **type inference** (вывод типа).
>
> **Когда писать тип явно?** Когда значение не даёт подсказки (например, пустой список `val tasks: List<Task> = emptyList()`) или когда хотите явно зафиксировать тип для читаемости.

### Почему типы важны?

Типы — это **защита от ошибок**. Представьте, что вы написали:

```kotlin
val age = "25"     // текст "25", не число 25!
val next = age + 1 // что произойдёт?
```

В JavaScript это сработает и даст `"251"` (текст «25» + число 1 = текст «251»). В Kotlin компилятор сразу скажет: **ошибка, нельзя сложить String и Int**. Вы увидите проблему до запуска программы.

Это называется **статическая типизация** — типы проверяются на этапе компиляции, а не в runtime. Kotlin — статически типизированный язык.

## 3. Функция — это «именованный исполняемый кусок кода»

Теперь главное. Представьте, что у вас есть программа из 1000 строк. Если всё написать подряд — это будет нечитаемая каша. **Функция** — способ разбить программу на **именованные блоки**, каждый из которых делает одну вещь.

Аналогия: **кнопка на пульте**. Вы не думаете о том, какие сигналы идут от пульта к телевизору, какие команды выполняет процессор телевизора. Вы нажимаете кнопку «громче» — громкость увеличивается. Кнопка — это **функция**: у неё есть имя («громче»), она что-то делает (увеличивает громкость), и вам не нужно знать, как именно.

### Анатомия функции

```kotlin
fun greet(name: String): String {
    val message = "Привет, $name!"
    return message
}
```

Разберём по частям:

- **`fun`** — ключевое слово, объявляющее функцию. Без него Kotlin не поймёт, что это функция.
- **`greet`** — имя функции. По этому имени вы будете её «вызывать» (нажимать кнопку).
- **`(name: String)`** — **параметры**. Это «входные данные» функции. Функция `greet` принимает один параметр `name` типа `String`. Когда будете вызывать `greet("Alice")`, значение `"Alice"` попадёт в `name`.
- **`: String`** (после скобок) — **тип возвращаемого значения**. Функция возвращает результат типа `String`. Если функция ничего не возвращает — пишется `: Unit` (или ничего, `Unit` по умолчанию).
- **`{ ... }`** — **тело функции**. Код, который выполняется при вызове.
- **`return message`** — оператор возврата. Берёт значение `message` и отдаёт его вызывающему коду.

### Вызов функции

```kotlin
val result = greet("Alice")  // вызов: передаём "Alice" как name
println(result)              // выведет "Привет, Alice!"

// Можно сразу использовать результат
println(greet("Bob"))        // выведет "Привет, Bob!"
```

Когда вы пишете `greet("Alice")`:
1. Компьютер **останавливает** текущий код.
2. **Переходит** в тело функции `greet`, подставляя `"Alice"` вместо `name`.
3. Выполняет код тела функции построчно.
4. Доходит до `return message` — берёт значение `message`.
5. **Возвращается** обратно в то место, где функция была вызвана, и подставляет результат.
6. Продолжает выполнение с этой точки.

### Параметры — это «входы» функции

Функция может принимать **сколько угодно параметров** (или ноль):

```kotlin
// Без параметров
fun sayHello() {
    println("Hello!")
}

// С двумя параметрами
fun add(a: Int, b: Int): Int {
    return a + b
}

// С тремя параметрами разных типов
fun createUser(name: String, age: Int, isActive: Boolean): User {
    return User(name, age, isActive)
}
```

Параметры — это **временные переменные**, которые существуют только пока функция выполняется. Когда функция завершается — параметры исчезают.

> [!TIP]
> **Параметры функции — как формальные аргументы в математике.** В математике пишут `f(x) = x² + 1`. Здесь `x` — формальный параметр. Когда вызываете `f(3)`, `x` принимает значение 3, и результат — 10. В Kotlin то же самое: `fun f(x: Int) = x * x + 1`. Вызов `f(3)` вернёт 10.

### Возвращаемое значение — это «выход» функции

Функция может **возвращать** результат (одно значение) или **ничего не возвращать** (`Unit`):

```kotlin
// Возвращает Int
fun multiply(a: Int, b: Int): Int {
    return a * b
}

// Ничего не возвращает (Unit)
fun log(message: String) {
    println("[LOG] $message")
    // нет return — функция возвращает Unit (пустота)
}

// Короткий синтаксис для однострочных функций
fun square(x: Int): Int = x * x
// Эквивалент:
// fun square(x: Int): Int { return x * x }
```

> [!TIP]
> **`Unit`** — это тип «пустоты» в Kotlin. Аналог `void` в Java/C++. Функция, которая ничего не возвращает, неявно возвращает `Unit`. Можно написать `fun log(msg: String): Unit { ... }` — это то же самое, что и без `: Unit`.
>
> **`return`** — оператор, который немедленно завершает функцию и отдаёт значение. После `return` код в теле функции не выполняется. Можно вернуть раньше, например, при ошибке:
> ```kotlin
> fun divide(a: Int, b: Int): Int {
>     if (b == 0) return 0  // ранний возврат
>     return a / b
> }
> ```

### Зачем функции?

1. **Повторное использование.** Написали один раз — вызываете хоть сто раз. `greet("Alice")`, `greet("Bob")`, `greet("Charlie")` — один и тот же код работает с разными данными.
2. **Разделение ответственности.** Вместо 1000 строк сплошного кода — 50 функций по 20 строк. Каждая функция делает одну вещь, у неё есть понятное имя.
3. **Абстракция.** Вызывая `getUserById(42)`, вы не думаете, как именно пользователь достаётся из базы. Функция скрывает детали — вы работаете с **что**, а не с **как**.
4. **Тестируемость.** Функцию можно протестировать отдельно от остального кода: вызвали с разными входами, проверили выходы.

### Функции высшего порядка — функции как данные

В Kotlin функции — это **значения**. Их можно передавать как параметры, возвращать из других функций, хранить в переменных. Это называется **функции высшего порядка** и это мощная концепция.

```kotlin
// Функция, принимающая другую функцию как параметр
fun applyOperation(x: Int, y: Int, operation: (Int, Int) -> Int): Int {
    return operation(x, y)  // вызываем переданную функцию
}

// Использование
val sum = applyOperation(5, 3) { a, b -> a + b }       // 8
val product = applyOperation(5, 3) { a, b -> a * b }   // 15
```

> [!TIP]
> **`(Int, Int) -> Int`** — это **тип функции**, которая принимает два `Int` и возвращает `Int`. Типы функций в Kotlin записываются так: `(типы_параметров) -> тип_возвращаемого_значения`.
>
> **`{ a, b -> a + b }`** — это **лямбда-выражение** (анонимная функция). Запись `{ args -> body }` создаёт функцию без имени. Лямбда ` { a, b -> a + b }` эквивалентна `fun(a: Int, b: Int): Int = a + b`, но короче и может быть передана прямо в месте вызова.

## 4. Класс — это «чертёж» для создания объектов

Теперь перейдём к более сложной концепции. Представьте, что вы проектируете дом. У вас есть **чертёж** — лист бумаги с описанием: дом имеет стены, крышу, окна, дверь. По одному чертежу можно построить **много домов** — каждый будет отдельным объектом, но все они следуют одному плану.

**Класс** — это чертёж. **Объект** (или **экземпляр**) — это конкретная реализация, построенная по этому чертежу.

```kotlin
// Чертёж: класс Dog описывает, что у каждой собаки есть имя и возраст
class Dog(val name: String, val age: Int) {
    fun bark() {
        println("$name says: Woof!")
    }
}

// Создаём конкретные объекты (экземпляры) по этому чертежу
val dog1 = Dog("Rex", 3)    // первая собака
val dog2 = Dog("Buddy", 5)  // вторая собака

dog1.bark()  // Rex says: Woof!
dog2.bark()  // Buddy says: Woof!

println(dog1.age)  // 3
println(dog2.age)  // 5
```

Разберём:

- **`class Dog`** — объявляем класс с именем `Dog`. Это **чертёж**.
- **`(val name: String, val age: Int)`** — **конструктор**. Описывает, какие данные нужны, чтобы создать собаку. Каждая собака будет иметь поля `name` и `age`.
- **`fun bark() { ... }`** — **метод** класса. Это функция, привязанная к классу. Каждая собака умеет лаять.
- **`Dog("Rex", 3)`** — **создание экземпляра**. Берём чертёж `Dog`, передаём `name = "Rex"` и `age = 3`, получаем конкретный объект.
- **`dog1.bark()`** — вызов метода на конкретном объекте. Компьютер знает, что `dog1` — это собака с именем "Rex", и подставляет это имя в `bark()`.

### Поля и методы — «состояние» и «поведение»

У объекта есть две составляющие:

| Термин | Что это | Аналогия |
|--------|---------|----------|
| **Поле** (property) | Данные внутри объекта | Возраст, имя, цвет собаки |
| **Метод** (method) | Действия, которые объект может выполнять | Лаять, бежать, есть |

```kotlin
class BankAccount(val owner: String, var balance: Double) {
    // Поле: owner (неизменяемое, val)
    // Поле: balance (изменяемое, var — баланс меняется со временем)

    // Метод: положить деньги
    fun deposit(amount: Double) {
        balance += amount
        println("$owner deposited $$amount. New balance: $$balance")
    }

    // Метод: снять деньги
    fun withdraw(amount: Double): Boolean {
        if (amount > balance) {
            println("Insufficient funds for $owner")
            return false
        }
        balance -= amount
        println("$owner withdrew $$amount. New balance: $$balance")
        return true
    }
}

// Использование
val account = BankAccount("Alice", 100.0)
account.deposit(50.0)    // Alice deposited $50. New balance: $150.0
account.withdraw(30.0)   // Alice withdrew $30. New balance: $120.0
account.withdraw(200.0)  // Insufficient funds for Alice
```

> [!TIP]
> **`val` vs `var` в полях класса:**
> - `val owner: String` — после создания объекта поле `owner` нельзя изменить (собака навсегда «Alice»).
> - `var balance: Double` — поле можно менять (баланс растёт и падает со временем).
>
> Это те же `val` и `var`, что и для обычных переменных — но применённые внутри класса они создают **поля объекта**.

### Конструктор — «способ создания»

**Конструктор** — это специальный код, который выполняется при создании объекта. В Kotlin основной конструктор объявляется прямо в заголовке класса:

```kotlin
class Task(val title: String, var completed: Boolean = false)
//      ^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^
//      Это и есть конструктор: принимает title и completed (по умолчанию false)

// Создание:
val task1 = Task("Buy milk")              // completed = false (по умолчанию)
val task2 = Task("Walk dog", true)        // completed = true (явно)
```

Можно добавить **тело конструктора** — код, выполняемый при создании:

```kotlin
class User(val name: String, val age: Int) {
    // init-блок выполняется при создании каждого объекта
    init {
        println("Created user $name, age $age")
    }
}

val u = User("Alice", 25)  // выведет "Created user Alice, age 25"
```

### Зачем классы?

1. **Группировка связанных данных и логики.** Вместо `name1, age1, name2, age2, ...` — один класс `Person`.
2. **Создание множества похожих объектов.** Один класс `Task` — тысяча конкретных задач.
3. **Инкапсуляция** — скрытие деталей. Внешний код не знает, как `BankAccount` хранит баланс, он просто вызывает `deposit()`.
4. **Типобезопасность.** Если функция принимает `Task`, компилятор не даст передать туда `Person` — разные чертежи.

## 5. Объект — «экземпляр класса»

Когда вы пишете `Dog("Rex", 3)`, происходит:
1. Компьютер **выделяет память** под новый объект.
2. **Вызывает конструктор** `Dog`, передавая `"Rex"` и `3`.
3. Конструктор **инициализирует поля**: `name = "Rex"`, `age = 3`.
4. Возвращает **ссылку** на созданный объект.
5. Эта ссылка записывается в переменную `dog1`.

```kotlin
val dog1 = Dog("Rex", 3)
val dog2 = Dog("Rex", 3)
val dog3 = dog1   // dog3 ссылается на тот же объект, что и dog1

dog1 == dog2      // false — это разные объекты в памяти (хотя и одинаковые)
dog1 === dog2     // false — разные ссылки
dog1 === dog3     // true — одна и та же ссылка
```

> [!TIP]
> **`==` vs `===` в Kotlin:**
> - `==` проверяет **равенство по значению** (вызывает `equals()`). Две собаки с одинаковыми именами и возрастом равны по значению, но это разные объекты.
> - `===` проверяет **равенство по ссылке** (это тот же самый объект в памяти). `dog1 === dog3` true, потому что `dog3 = dog1` — обе переменные указывают на один объект.

### Классы с авто-сгенерированными методами: `data class`

Обычный класс `class Task(...)` не имеет автоматически сгенерированных `equals()`, `hashCode()`, `toString()`. Если хотите, чтобы два объекта с одинаковыми полями считались равными — используйте `data class`:

```kotlin
data class Task(val title: String, val completed: Boolean = false)

val t1 = Task("Buy milk")
val t2 = Task("Buy milk")
println(t1 == t2)  // true — data class сравнивает по полям
println(t1)        // Task(title=Buy milk, completed=false) — красивый toString
```

> [!TIP]
> **`data class`** — это класс, для которого Kotlin автоматически генерирует методы `equals()`, `hashCode()`, `toString()`, `copy()`, `componentN()`. Используется для классов, главная задача которых — **хранить данные** (DTO, модели, значения). Для классов с поведением (например, `BankAccount` с логикой deposit/withdraw) обычный `class` предпочтительнее.

## 6. Условия — «выбор пути»

Программа редко идёт по прямой линии. Часто нужно **разветвлять**: если пользователь залогинен — показать ленту, иначе — экран входа. Для этого есть **условные операторы**.

### `if` — простое условие

```kotlin
val age = 20

if (age >= 18) {
    println("Взрослый")
} else if (age >= 13) {
    println("Подросток")
} else {
    println("Ребёнок")
}
// Выведет "Взрослый"
```

- **`if (условие)`** — если условие истинно (`true`), выполнится блок в `{ }`.
- **`else if`** — иначе, если другое условие истинно...
- **`else`** — иначе (если ни одно из условий не сработало).

В Kotlin `if` — это **выражение**, оно возвращает значение:

```kotlin
val status = if (age >= 18) "Взрослый" else "Ребёнок"
// status = "Взрослый"
```

### `when` — мощный аналог switch

Когда условий много, `when` читается лучше:

```kotlin
val dayOfWeek = 3
val dayName = when (dayOfWeek) {
    1 -> "Понедельник"
    2 -> "Вторник"
    3 -> "Среда"
    4 -> "Четверг"
    5 -> "Пятница"
    6, 7 -> "Выходной"          // несколько значений через запятую
    else -> "Неизвестный день"   // если ничего не совпало
}
// dayName = "Среда"
```

> [!TIP]
> **`when`** — это «умный switch». В отличие от switch в Java/C, в Kotlin `when`:
> - Возвращает значение (можно присвоить переменной).
> - Не нужно писать `break` после каждой ветки (как в Java).
> - Может проверять не только равенство, но и любые условия: `is String ->`, `in 1..10 ->`, и т.д.
> - Если `when` используется как выражение, обязателен `else` (если не все случаи покрыты).

## 7. Циклы — «повторение»

Часто нужно повторить одно и то же действие много раз. Например, вывести числа от 1 до 10, или обработать каждый элемент списка.

### `for` — цикл по диапазону или коллекции

```kotlin
// Перебор чисел от 1 до 5
for (i in 1..5) {
    println(i)
}
// Выведет: 1 2 3 4 5

// Перебор элементов списка
val fruits = listOf("apple", "banana", "cherry")
for (fruit in fruits) {
    println(fruit)
}
// Выведет: apple banana cherry

// С индексами
for ((index, fruit) in fruits.withIndex()) {
    println("$index: $fruit")
}
// Выведет: 0: apple, 1: banana, 2: cherry
```

### `while` — цикл с условием

```kotlin
var attempts = 0
while (attempts < 3) {
    println("Попытка ${attempts + 1}")
    attempts++
}
// Выведет: Попытка 1, Попытка 2, Попытка 3
```

`while (условие)` — пока условие истинно, выполнять блок. **Важно**: условие проверяется перед каждой итерацией, и если оно сразу ложно — блок не выполнится ни разу.

> [!TIP]
> **`for` vs `while` — когда что:**
> - `for` — когда знаете, сколько раз повторить (перебрать список, диапазон 1..10).
> - `while` — когда повторяете, пока выполняется условие, и не знаете заранее, сколько итераций понадобится (ждать ответа от сервера, искать элемент).

## 8. Коллекции — «контейнеры для множества значений»

Переменная хранит одно значение. Что если нужно хранить **много** значений? Например, список задач, множество уникальных тегов, словарь «пользователь → email». Для этого есть **коллекции**.

### `List` — упорядоченный список

```kotlin
val tasks = listOf("Buy milk", "Walk dog", "Read book")

println(tasks[0])         // Buy milk (доступ по индексу, с 0)
println(tasks.size)       // 3

for (task in tasks) {
    println("- $task")
}

// Изменяемый список
val mutableTasks = mutableListOf("Task 1")
mutableTasks.add("Task 2")     // добавляем
mutableTasks.remove("Task 1")  // удаляем
mutableTasks[0] = "Updated"    // заменяем
```

### `Set` — множество уникальных элементов

```kotlin
val tags = setOf("kotlin", "android", "kotlin", "mobile")
// Дубликаты автоматически удаляются
println(tags)  // [kotlin, android, mobile]
println(tags.size)  // 3 (не 4!)

// Проверка принадлежности
println("kotlin" in tags)  // true
```

### `Map` — словарь «ключ → значение»

```kotlin
val userAges = mapOf(
    "Alice" to 25,
    "Bob" to 30,
    "Charlie" to 35
)

println(userAges["Alice"])  // 25 (доступ по ключу)
println(userAges.size)      // 3

// Изменяемый Map
val mutable = mutableMapOf<String, Int>()
mutable["Dave"] = 40
mutable["Alice"] = 26  // обновляем
```

> [!TIP]
> **Иммутабельные vs мутабельные коллекции:**
> - `listOf`, `setOf`, `mapOf` — **неизменяемые** (immutable). Нельзя добавить или удалить элемент после создания. Это безопаснее: можно передавать в функции, не боясь, что они изменят коллекцию.
> - `mutableListOf`, `mutableSetOf`, `mutableMapOf` — **изменяемые** (mutable). Можно добавлять, удалять, обновлять элементы.
>
> В Kotlin принято: **сначала попробуйте immutable**. Если действительно нужно менять — используйте mutable. Большую часть времени immutable достаточно.

## 9. Null — «пустота»

В программировании есть особое значение `null` — «ничего», «пусто», «значение отсутствует». Например, пользователь не заполнил поле email — там `null`. Функция не нашла элемент — возвращает `null`.

В Kotlin типы по умолчанию **не могут быть null**:

```kotlin
var name: String = "Alice"
name = null  // ❌ ОШИБКА КОМПИЛЯЦИИ — String не может быть null

// Чтобы разрешить null, добавьте ? в тип
var nullableName: String? = "Alice"
nullableName = null  // ✅ ОК
```

> [!TIP]
> **`?` в типе** (`String?`) означает «этот тип может быть null». `String` (без `?`) — точно не null, компилятор это гарантирует. `String?` — может быть null, нужно проверять перед использованием.
>
> Это одна из главных фич Kotlin — **null safety на уровне типа**. В Java любая ссылка может быть null, что приводит к миллионам `NullPointerException`. В Kotlin компилятор заставляет явно обрабатывать nullable-типы.

### Безопасная работа с null

```kotlin
val name: String? = getName()

// 1. Safe call ?.
val length: Int? = name?.length  // если name не null — вернёт длину, иначе null

// 2. Elvis operator ?:
val safeLength: Int = name?.length ?: 0  // если null — возьмём 0

// 3. Not-null assertion !! (опасно!)
val forced: Int = name!!.length  // если name null — бросит NullPointerException
// Используйте !! только когда на 100% уверены, что не null

// 4. let — выполнить блок, только если не null
name?.let {
    println("Имя: $it, длина: ${it.length}")
}
```

## 10. Наследование — «расширение класса»

Класс может **наследовать** от другого класса — получить все его поля и методы, и добавить свои. Это как «базовый чертёж» и «расширенный чертёж».

```kotlin
// Базовый класс
open class Animal(val name: String) {
    fun eat() {
        println("$name is eating")
    }
}

// Производный класс — наследует от Animal
class Dog(name: String) : Animal(name) {
    fun bark() {
        println("$name says: Woof!")
    }
}

class Cat(name: String) : Animal(name) {
    fun meow() {
        println("$name says: Meow!")
    }
}

// Использование
val dog = Dog("Rex")
dog.eat()   // унаследованный метод — "Rex is eating"
dog.bark()  // свой метод — "Rex says: Woof!"

val cat = Cat("Whiskers")
cat.eat()   // унаследованный
cat.meow()  // свой
```

> [!TIP]
> **`open class`** — в Kotlin классы по умолчанию **закрыты для наследования** (финальные). Чтобы от класса можно было наследовать, нужно пометить его `open`. Это решение JetBrains, чтобы избежать хаоса с наследованием, как в Java (где любой класс можно унаследовать).
>
> **`class Dog(name: String) : Animal(name)`** — синтаксис наследования. `Dog` наследует от `Animal`, передавая `name` в конструктор базового класса.

### Переопределение методов

Производный класс может **переопределить** метод базового:

```kotlin
open class Animal(val name: String) {
    open fun sound() {
        println("$name makes a sound")
    }
}

class Dog(name: String) : Animal(name) {
    override fun sound() {
        println("$name says: Woof!")
    }
}

val animal: Animal = Dog("Rex")
animal.sound()  // "Rex says: Woof!" — вызовется Dog.sound(), не Animal.sound()
```

Это называется **полиморфизм**: переменная типа `Animal` может указывать на `Dog`, и вызов `sound()` вызовет правильную реализацию в зависимости от реального типа объекта.

## 11. Интерфейс — «контракт»

Интерфейс — это **описание того, что класс умеет делать**, без реализации. Класс **реализует** интерфейс, обязуясь предоставить код для всех его методов.

Аналогия: **договор на работу**. В договоре написано «сотрудник должен уметь писать код, тестировать, деплоить». Это интерфейс. Конкретный программист (класс) реализует этот интерфейс — он умеет делать всё это, и неважно, на каком языке он программирует.

```kotlin
// Интерфейс — контракт
interface Drawable {
    fun draw()           // без тела — класс должен реализовать
    fun area(): Double   // тоже без тела
}

// Класс Circle реализует Drawable
class Circle(val radius: Double) : Drawable {
    override fun draw() {
        println("Рисую круг радиусом $radius")
    }

    override fun area(): Double {
        return 3.14 * radius * radius
    }
}

// Класс Square тоже реализует Drawable
class Square(val side: Double) : Drawable {
    override fun draw() {
        println("Рисую квадрат со стороной $side")
    }

    override fun area(): Double {
        return side * side
    }
}

// Использование: работаем через интерфейс
fun render(shape: Drawable) {  // принимает любой Drawable
    shape.draw()
    println("Площадь: ${shape.area()}")
}

render(Circle(5.0))   // Рисую круг... Площадь: 78.5
render(Square(4.0))   // Рисую квадрат... Площадь: 16.0
```

> [!TIP]
> **Интерфейс vs абстрактный класс:**
> - Интерфейс — только контракт (методы без реализации, кроме `default` методов). Класс может реализовать **сколько угодно** интерфейсов.
> - Абстрактный класс — может содержать реализацию + абстрактные методы. Класс может наследовать только **один** абстрактный класс.
>
> В Kotlin интерфейсы тоже могут содержать методы с реализацией (default methods), но поля они хранить не могут. Если нужно共享 состояние — используйте абстрактный класс.

## 12. Корутины — «паузы в выполнении»

Обычная функция работает **синхронно**: вызвала — ждёте, пока закончится — получили результат. Но что если функция ждёт ответа от сервера (это может занять секунды)? Блокировать весь интерфейс на это время нельзя.

**Корутина** — это функция, которая может **приостанавливаться** (suspend), освобождая поток для других задач, и **возобновляться** потом. Аналогия: вы заказали пиццу. Пока её готовят, вы не стоите столбом у печи — занимаетесь своими делами. Когда пицца готова — вам звонят, вы её забираете.

```kotlin
// Обычная функция — блокирует поток
fun fetchUserBlocking(): User {
    Thread.sleep(2000)  // ждём 2 секунды
    return User("Alice")
}

// Suspend-функция — не блокирует поток
suspend fun fetchUserAsync(): User {
    delay(2000)  // приостанавливаем корутину на 2 секунды, поток свободен
    return User("Alice")
}

// Вызов корутины
suspend fun main() {
    println("Start: ${Date()}")
    val user = fetchUserAsync()  // не блокирует
    println("User: $user, time: ${Date()}")
}
```

> [!TIP]
> **`suspend`** — ключевое слово, помечающее функцию как корутину. Такая функция может вызывать другие `suspend`-функции (например, `delay()`) и сама может быть вызвана только из корутины или другой suspend-функции.
>
> **`delay(2000)`** vs **`Thread.sleep(2000)`**: `Thread.sleep` блокирует поток (никакой другой код не выполнится в этом потоке 2 секунды). `delay` приостанавливает **корутину**, но поток свободен выполнять другие корутины — эффективнее.
>
> Корутины описаны подробно в [главе 20. Глубокое погружение в Kotlin](20-deep-dive-kotlin.md#202-корутины-и-структурированная-конкурентность).

## 13. Общая картина

Теперь у вас есть фундамент. Краткая карта концепций:

| Концепция | Что это | Аналогия |
|-----------|---------|----------|
| **Переменная** | Именованная ячейка памяти | Коробка с этикеткой |
| **Тип данных** | Контракт о том, что в переменной | Ярлык на коробке |
| **Функция** | Именованный блок кода | Кнопка на пульте |
| **Параметр** | Вход функции | То, что вы передаёте кнопке |
| **Возвращаемое значение** | Выход функции | Результат нажатия кнопки |
| **Класс** | Чертёж для создания объектов | Архитектурный план |
| **Объект** | Конкретный экземпляр класса | Построенный по плану дом |
| **Поле** | Данные объекта | Свойства дома (стены, окна) |
| **Метод** | Действие объекта | Что дом «умеет» (открывать дверь) |
| **Конструктор** | Способ создания объекта | Процесс строительства |
| **Наследование** | Расширение класса | Дом + бассейн = особый дом |
| **Интерфейс** | Контракт | Договор с работником |
| **Условие (`if`/`when`)** | Выбор пути | Развилка на дороге |
| **Цикл (`for`/`while`)** | Повторение | Бег по кругу |
| **Коллекция** | Контейнер для множества значений | Полка с книгами |
| **Null** | Пустота | Пустая коробка |
| **Корутина** | Приостанавливаемая функция | Заказ пиццы с ожиданием |
