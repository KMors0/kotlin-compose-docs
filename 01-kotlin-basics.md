# Основы Kotlin

> Kotlin — современный статически типизированный язык программирования, работающий на JVM (Java Virtual Machine) и в JavaScript. Официально поддерживается Google для разработки Android-приложений, но также используется в бэкенде, кроссплатформенной разработке и других областях.

**Ключевые преимущества:**
- лаконичный синтаксис;
- совместимость с Java;
- безопасность нулевых значений (`null safety`);
- поддержка функционального программирования.

---

## 1.1. Что такое Kotlin?

Kotlin был разработан компанией JetBrains и впервые представлен в 2011 году. В 2017 году Google объявил Kotlin официальным языком для разработки Android-приложений, что стало мощным стимулом для его массового внедрения. Язык сочетает в себе объектно-ориентированные и функциональные возможности, предлагая при этом лаконичный и выразительный синтаксис, который устраняет большую часть шаблонного кода, характерного для Java.

Основная философия Kotlin — прагматизм. Язык создавался для решения реальных промышленных задач, а не для академических экспериментов. Это означает, что каждая конструкция в Kotlin имеет чёткое практическое применение, а компилятор активно помогает разработчику избегать типичных ошибок, таких как `NullPointerException`.

## 1.2. Базовые конструкции

- **Переменные и константы:** `val` (неизменяемая), `var` (изменяемая).
- **Типы данных:** `Int`, `String`, `Boolean`, `Float`, `Double`, коллекции (`List`, `Map`, `Set`).
- **Функции:** объявляются через `fun`, поддерживают лямбды и расширения.
- **Классы и объекты:** `data class` для простых структур данных, `sealed class` для ограниченных иерархий.
- **Условия и циклы:** `if`, `when`, `for`, `while`.
- **Корутины** (`kotlinx.coroutines`) — для асинхронного программирования.

**Пример простой функции:**
```kotlin
fun greet(name: String): String {
    return "Hello, $name!"
}
```

**Пример data class:**
```kotlin
data class User(val name: String, val age: Int)

val user = User("Alice", 30)
println(user.copy(age = 31)) // User(name=Alice, age=31)
```

## 1.3. Коллекции и функции высшего порядка

Kotlin предлагает мощные инструменты для работы с коллекциями:
- `map()`, `filter()`, `reduce()` — для преобразования данных.
- `forEach()` — для итерации.
- `listOf()`, `setOf()`, `mapOf()` — фабричные функции.

**Пример:**
```kotlin
val numbers = listOf(1, 2, 3, 4, 5)
val evenSquares = numbers.filter { it % 2 == 0 }.map { it * it }
println(evenSquares) // [4, 16]
```

**Функции высшего порядка** позволяют принимать другие функции в качестве аргументов или возвращать их. Это основа функционального стиля программирования в Kotlin. Благодаря лямбда-выражениям и inline-функциям, использование функциональных конструкций не создаёт накладных расходов на производительность.

**Пример функции высшего порядка:**
```kotlin
fun <T> List<T>.customFilter(predicate: (T) -> Boolean): List<T> {
    val result = mutableListOf<T>()
    for (item in this) {
        if (predicate(item)) result.add(item)
    }
    return result
}
```

## 1.4. Null Safety

В Kotlin нельзя случайно обратиться к `null`:
- `?` — обозначение возможного `null`-значения.
- `!!` — принудительное приведение (опасно, может вызвать `NullPointerException`).
- `?.` и `?:` — безопасные вызовы и оператор Элвиса.

**Пример:**
```kotlin
val name: String? = null
val length = name?.length ?: 0 // длина = 0, если name == null
```

> [!TIP]
> **`?.` (safe call operator)** — «если слева не null, вызови метод; иначе верни null». Эквивалент `if (name != null) name.length else null`. Результат `name?.length` имеет тип `Int?` (тоже nullable).
>
> **`?:` (Elvis operator)** — «если слева не null, возьми его; иначе возьми то, что справа». Назван в честь Элвиса Пресли (похоже на прическу `?:` ↪). Эквивалент `if (left != null) left else right`. Часто используется для задания значения по умолчанию.

Система null safety в Kotlin — одна из самых важных особенностей языка. В отличие от Java, где любая ссылка может быть `null`, Kotlin разделяет типы на nullable и non-nullable на уровне системы типов. Это означает, что компилятор принудительно требует обработки возможных `null`-значений, что значительно снижает количество `NullPointerException` в runtime.

**Расширенные примеры:**
```kotlin
// Безопасный вызов с let
val name: String? = getName()
name?.let {
    println("Длина имени: ${it.length}")
}

// Приведение типа с безопасностью
val result: String? = data as? String
```

> [!TIP]
> **`let`** — встроенная функция Kotlin, принимает лямбду и вызывает её с объектом, на котором вызвана. Синтаксис `receiver.let { lambda }` эквивалентен `lambda(receiver)`. Используется для safe-call: `name?.let { ... }` означает «если `name` не null, выполни блок с `name` как аргументом».
>
> **`it`** — автоматическое имя единственного параметра лямбды. Внутри `name?.let { println(it.length) }` лямбда принимает один параметр (в данном случае `name`), и Kotlin автоматически называет его `it`. Можно дать своё имя: `name?.let { value -> println(value.length) }` — но `it` короче и привычнее для коротких лямбд.
>
> **`as?` (safe cast)** — «если приведение возможно, верни результат; иначе верни null». Эквивалент `if (data is String) data else null`. В отличие от обычного `as` (который бросает `ClassCastException` при неудаче), `as?` безопасен.

### Scope-функции: let, run, with, apply, also

В Kotlin есть 5 встроенных функций, которые работают с объектом в контексте блока кода. Они отличаются тем, как получают объект и что возвращают:

| Функция | Объект доступен как | Возвращает | Типичное использование |
|---------|---------------------|------------|------------------------|
| `let` | `it` (параметр) | Результат лямбды | Преобразование, null-checks |
| `run` | `this` (получатель) | Результат лямбды | Вычисления над объектом |
| `with` | `this` (получатель) | Результат лямбды | Группировка вызовов на объекте |
| `apply` | `this` (получатель) | Сам объект | Инициализация/конфигурация |
| `also` | `it` (параметр) | Сам объект | Side-effects (логирование, валидация) |

**Примеры:**
```kotlin
// let — преобразование и null-check
val length: Int? = name?.let { it.length }

// apply — конфигурация объекта (возвращает сам объект)
val person = Person().apply {
    name = "Alice"        // внутри apply this = person
    age = 30              // можно обращаться к полям без префикса
    email = "alice@example.com"
}

// also — side-effect без изменения цепочки
val result = computeValue()
    .also { println("Получено: $it") }  // логирование
    .also { validate(it) }              // валидация
    .let { transform(it) }              // преобразование

// run — вычисление с контекстом объекта
val greeting = person.run {
    // this = person
    "Привет, $name! Тебе $age лет."
}

// with — группировка вызовов (не extension)
with(person) {
    println(name)
    println(age)
    println(email)
}
```

> [!TIP]
> **`this` vs `it` внутри scope-функций:** когда функция делает объект `this` (как `apply`, `run`, `with`), вы обращаетесь к полям объекта без префикса — `name` вместо `person.name`. Когда функция делает объект параметром `it` (как `let`, `also`), вы обращаетесь через `it.name`.
>
> **Возвращаемое значение:** `apply` и `also` возвращают сам объект — удобно для chain-вызовов (`obj.apply {...}.also {...}`). `let`, `run`, `with` возвращают результат последнего выражения в лямбде — удобно для преобразований.
>
> **Мнемоника для запоминания:** функции с `a` (`apply`, `also`) возвращают сам объект (как буква `a` похожа на петлю, замыкающуюся на себя). Функции без `a` (`let`, `run`, `with`) возвращают результат лямбды.
