# Конкурентный ландшафт: KMP vs Flutter vs React Native

> Объективное сравнение трёх основных кроссплатформенных решений для принятия обоснованного решения о выборе технологического стека.

---

## Сравнительная таблица

| Критерий | Kotlin Multiplatform + Compose | Flutter | React Native |
|----------|-------------------------------|---------|--------------|
| **Язык** | Kotlin | Dart | JavaScript/TypeScript |
| **UI-подход** | Декларативный, Skia-рендеринг | Декларативный, Skia-рендеринг | Декларативный, нативные компоненты |
| **iOS-производительность** | Высокая (Skia/Metal), stable с 1.8.0 | Высокая (Impeller/Skia), production-ready | Высокая (JSI + нативные мосты), production-ready |
| **Web-поддержка** | Beta (Kotlin/Wasm) | Stable (Canvas/HTML) | Stable (React DOM) |
| **Desktop-поддержка** | Stable (JVM/Skia) | Stable (Win/Mac/Linux) | Ограниченная (via react-native-windows/macOS) |
| **Динамический код** | Нет (статическая компиляция) | Нет (AOT/JIT) | Да (CodePush, обновления без App Store) |
| **Интероперабельность** | Полная с Java/Kotlin, нативная с Swift/ObjC через cinterop | Нативные каналы (Platform Channels) | Нативные модули |
| **Экосистема** | Растущая, KMP-focused | Зрелая, обширная pub.dev | Очень зрелая, npm-экосистема |
| **Подход к коду** | Общая логика + общая UI (опционально) | Общая логика + общая UI (обязательно) | Общая логика + общая UI (обязательно) |
| **Enterprise-adopters** | Netflix, McDonald's, Cash App, Google Workspace | BMW, Google, Alibaba, Nubank | Meta, Microsoft, Shopify |

---

## Ключевое дифференцирующее преимущество KMP

**Гибкость** — можно делить только бизнес-логику, оставляя нативный UI (gradual adoption), или делить и логику, и UI (Compose Multiplatform). Flutter и React Native требуют полного отказа от нативного UI, что может быть неприемлемо для продуктов, где нативный look-and-feel критичен.

По данным survey JetBrains, KMP вырос с 7% до 18% использования за два года, а по данным LinkedIn-публикаций — доля KMP в enterprise-сегменте удвоилась за 18 месяцев. Это указывает на растущий тренд, хотя Flutter и React Native по-прежнему занимают бóльшую долю рынка в абсолютных цифрах.

---

## Когда выбирать KMP + Compose

- Команда уже имеет сильную Kotlin/Android-экспертизу
- Нужно инкрементальное внедрение (не переписывать всё сразу)
- Критична нативная производительность на iOS
- Необходима десктопная поддержка
- Важно сохранить возможность использования нативных UI-компонентов

## Когда выбирать Flutter

- Команда готова полностью перейти на Dart
- Нужна зрелая веб-поддержка прямо сейчас
- Важна обширная экосистема готовых пакетов
- Приложение не требует глубокой интеграции с платформенными API

## Когда выбирать React Native

- Команда имеет сильную JavaScript/TypeScript-экспертизу
- Критична возможность обновления без App Store (CodePush)
- Нужна максимальная переиспользуемость веб-компонентов
- Приложение в основном работает с веб-API и простыми списками

---

## Рекомендация для организаций

Для команд с сильной Kotlin/Android-экспертизой, стремящихся к кроссплатформенности без потери нативного качества, стек Kotlin + Compose Multiplatform + Material 3 является оптимальным выбором в 2025–2026 годах. Для проектов с критичными веб-требованиями (SEO, контентные платформы) рекомендуется рассмотреть гибридный подход: Compose Multiplatform для мобильных и десктопа + специализированный веб-фреймворк для веба.
